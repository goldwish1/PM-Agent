# 决策记录（Decision Record）可起草工具 — 需求文档

## 1. 背景

当前 pmbox 支持「项目章程」和「风险登记册」两种可起草+导出文档。用户在实际项目管理中常遇到「需要做决策、权衡方案」的场景——和 AI 讨论梳理思路后，希望 AI 能帮忙归档成一份结构化的决策记录文档，而不是只留在对话历史里。

## 2. 目标

新增 **决策记录（Decision Record）** 可起草工具，复用现有「陪跑咨询 → 起草 → 确认 → 导出」闭环，使用户能够：

1. 与 AI 讨论决策场景（如选方案、权衡 trade-off）
2. AI 基于讨论内容提炼决策记录字段
3. 预览并确认后导出为 Markdown 文档

## 3. 字段设计

参考架构决策记录（ADR / Decision Log）模式，设计以下字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| decision_title | str | 决策标题（要决定什么） |
| context | str | 背景/问题陈述 |
| options_considered | str | 考虑的备选方案（多行文本，类比 milestones 字段） |
| decision | str | 最终决定 |
| rationale | str | 决策依据/权衡理由 |
| consequences | str | 预期影响/后果（含风险） |
| decision_maker | str | 决策人 |
| decision_date | str | 决策日期 |
| status | str | 状态（拟定/已生效/已废弃） |

## 4. 需改动的文件

| # | 文件 | 改动内容 |

## 5. 体验流程

```
用户: "我在纠结用自研还是外采方案..."
  → AI 调用 start_consulting 进入陪跑模式
  → 对话中调用 note_consulting_fact 沉淀事实
  → 讨论充分后 AI 询问是否起草决策记录
  → 用户同意 → AI 调用 draft_decision_record 提炼字段
  → 展示预览 → 用户确认 → 调用 export_markdown(doc_type=decision)
  → 输出 output/决策记录-20260717-1420.md
```

## 6. 不做的事

- 不做多个决策记录的管理（MVP 一次只起草一条）
- 不做 options_considered 的列表结构（用多行文本兜底）
- 不修改路径白名单机制
